Where to Intervene? First Deposit vs. Second Deposit

Funnel math for the “$10 first purchase on us” experiment
DRI: Justin Zhou  •  Data: Crimson joiners Oct 2025–Mar 2026  •  Pulled: Apr 29, 2026
Verdict
First-deposit intervention produces 25% more durable users in absolute terms; second-deposit intervention is 20% cheaper per durable user.

A 10% relative lift on first-deposit conversion produces +467 durables per 100K joiners. The same lift on second-deposit conversion (among existing first-depositors) produces +375 durables. But because second-deposit triggers fewer payouts, B is more efficient per dollar ($255/durable vs $305/durable assuming $10/conversion).

1 The question

The experiment offers $10 off the first card purchase after an outside deposit. The same dollar could instead reward a second deposit. Per Nancy: “The math will tell you. It’s OK to pick one, but make it informed.”

Frame: Apply a 10% relative lift at one funnel step at a time, hold the rest of the funnel constant, and count incremental durable users.

Why A wins on volume: intervention A pulls in entirely new first-depositors. Each one carries the full downstream funnel. So +1,293 first depositors → +870 second depositors → +437 durables, plus +30 from first-only durables = +467 durables.

6 Recommendation

Both interventions miss the $210/durable target, so neither is unambiguously a winner. The phone-bill experiment baseline of $392/durable (best cohort) makes either of these look favorable. Pick based on what you optimize for:

If maximizing total impact at scale → A (first deposit)

A's addressable pool is 7.7× larger (100K joiners vs. 13K first-depositors). For the same intervention duration, A produces +467 durables per 100K joiners while B produces +375. The first-deposit experiment is already designed and ready to launch — this is the path of least resistance.

If maximizing $-efficiency → B (second deposit)

If budget is the constraint, B is 19.5% more efficient. Same $10K produces 6 more durables. B's per-incremental-durable cost ($255) is closer to the $210 target than A's ($305).

My take

Run A first, but instrument B. The first-deposit offer is already approved and on track for May 1–15. After observing results, the natural next test is: does adding a second-deposit incentive on top compound the effect? The math suggests yes — the durability lever is most concentrated at the second deposit.

If A's actual lift comes in below the modeled +10% (e.g., the phone-bill experiment achieved +6.5% relative on the cleanest cohort, not 10%), the calculus may shift toward B. Re-run this comparison post-A-readout with the actual measured lift instead of the assumed 10%.

7 Caveats
  • 10% lift is assumed equal at both steps. In practice, lifts may differ by step. Ancillary money-in is harder to move than card spend (the phone-bill experiment achieved +9.5% on first-time-payer adoption; getting that on first-deposit conversion is uncertain).
  • Cost per conversion ($10) is identical by assumption. Real offer mechanics could differ — e.g., a first-purchase reward vs. a flat second-deposit bonus may have different costs.
  • Durability is measured lifetime on the cohort window, not within a fixed promo month. A monthly definition would lower both rates but the relative comparison should hold.
  • Independent funnel rates assumed. The model assumes new-first-depositors converted by intervention A behave like organic first-depositors downstream. The phone-bill experiment showed marginal users were slightly less engaged than organic ones, so this likely overstates A's downstream durability.
  • No double-counting of behavior change. If intervention A changes who gets activated (e.g., pulls in less-engaged users), their second-deposit and durability rates may be lower than the baseline 67% and 50%.
8 SQL queries (for QC)

All numbers above pulled from this single query against edw.payments.fact_crimson_transactions.

Funnel query: joiners → 1st deposit → 2nd deposit → durable 
WITH payout_dasher AS (
  SELECT
    pa.id AS payout_account_id,
    dd.dasher_id
  FROM payout_service.public.payout_accounts pa
  JOIN proddb.public.dimension_dasher dd
    ON dd.payment_account_id = pa.payment_account_id
  WHERE dd.dasher_id IS NOT NULL
),

-- Crimson "join" = first-ever activity in the fact table for that payout account
crimson_joiners AS (
  SELECT
    pd.dasher_id,
    MIN(CAST(t.extract_datetime2 AS TIMESTAMP_NTZ)) AS join_ts
  FROM edw.payments.fact_crimson_transactions t
  JOIN payout_dasher pd ON t.payout_account_id = pd.payout_account_id
  WHERE t.extract_datetime2 IS NOT NULL
  GROUP BY 1
),

joiners_in_window AS (
  SELECT *
  FROM crimson_joiners
  WHERE CAST(join_ts AS DATE) BETWEEN DATE '2025-10-01' AND DATE '2026-03-31'
),

-- Ancillary money-in (excludes IN_DXPAY = DoorDash earnings)
ancillary_in AS (
  SELECT
    pd.dasher_id,
    CAST(t.extract_datetime2 AS TIMESTAMP_NTZ) AS deposit_ts,
    ABS(t.transaction_amount_with_direction) AS amt
  FROM edw.payments.fact_crimson_transactions t
  JOIN payout_dasher pd ON t.payout_account_id = pd.payout_account_id
  WHERE t.txn_code_type IN ('IN_ATM','IN_AFT','IN_ACH','IN_OCT')
    AND t.extract_datetime2 IS NOT NULL
),

-- Rank each user's deposits 1..N (only deposits AFTER they joined Crimson)
deposits_with_rank AS (
  SELECT
    a.dasher_id,
    a.deposit_ts,
    a.amt,
    ROW_NUMBER() OVER (PARTITION BY a.dasher_id ORDER BY a.deposit_ts) AS rn,
    SUM(a.amt) OVER (PARTITION BY a.dasher_id) AS lifetime_anc
  FROM ancillary_in a
  JOIN joiners_in_window j ON j.dasher_id = a.dasher_id
  WHERE a.deposit_ts >= j.join_ts
),

deposit_summary AS (
  SELECT
    dasher_id,
    MAX(CASE WHEN rn = 1 THEN deposit_ts END) AS first_dep_ts,
    MAX(CASE WHEN rn = 2 THEN deposit_ts END) AS second_dep_ts,
    MAX(rn)                                   AS total_deposits,
    MAX(lifetime_anc)                         AS lifetime_anc
  FROM deposits_with_rank
  GROUP BY 1
)

SELECT
  COUNT(*)                                    AS n_joiners,
  COUNT(ds.first_dep_ts)                      AS n_first_depositors,
  COUNT(ds.second_dep_ts)                     AS n_second_depositors,
  SUM(CASE WHEN ds.lifetime_anc >= 200
            THEN 1 ELSE 0 END)                AS n_ever_durable,

  -- Durable conditional on funnel position
  SUM(CASE WHEN ds.first_dep_ts IS NOT NULL
            AND ds.lifetime_anc >= 200
            THEN 1 ELSE 0 END)                AS n_first_then_durable,
  SUM(CASE WHEN ds.second_dep_ts IS NOT NULL
            AND ds.lifetime_anc >= 200
            THEN 1 ELSE 0 END)                AS n_second_then_durable,
  SUM(CASE WHEN ds.first_dep_ts IS NOT NULL
            AND ds.second_dep_ts IS NULL
            AND ds.lifetime_anc >= 200
            THEN 1 ELSE 0 END)                AS n_first_only_durable,
  SUM(CASE WHEN ds.first_dep_ts IS NOT NULL
            AND ds.second_dep_ts IS NULL
            THEN 1 ELSE 0 END)                AS n_first_only

FROM joiners_in_window jw
LEFT JOIN deposit_summary ds ON jw.dasher_id = ds.dasher_id;

Lift math (Python pseudo-code) 
p1            = 0.12931   # P(1st | joiner)
p2_given_1    = 0.67328   # P(2nd | 1st)
d_given_2     = 0.50133   # P(durable | 2nd)
d_given_1only = 0.06994   # P(durable | 1st only)

# Per 100,000 joiners, baseline:
joiners        = 100_000
first          = joiners * p1                        # 12,931
second         = first   * p2_given_1                # 8,707
first_only     = first   * (1 - p2_given_1)          # 4,224
durable_2nd    = second  * d_given_2                 # 4,365
durable_1only  = first_only * d_given_1only          # 295
durable_total  = durable_2nd + durable_1only         # 4,660

# Intervention A: 10% lift on p1
p1_A           = p1 * 1.10                           # 0.14224
first_A        = joiners * p1_A                      # 14,224
second_A       = first_A * p2_given_1                # 9,577
first_only_A   = first_A * (1 - p2_given_1)          # 4,646
durable_A      = second_A * d_given_2 + first_only_A * d_given_1only
                                                     # 5,127
delta_A        = durable_A - durable_total           # +467

# Intervention B: 10% lift on p2_given_1 (first-depositors only)
p2_B           = p2_given_1 * 1.10                   # 0.74061
second_B       = first * p2_B                        # 9,577
first_only_B   = first - second_B                    # 3,354
durable_B      = second_B * d_given_2 + first_only_B * d_given_1only
                                                     # 5,036
delta_B        = durable_B - durable_total           # +375

# Cost @ $10/conversion:
cost_A   = first_A  * 10        # $142,240
cost_B   = second_B * 10        # $95,770
$_per_durable_A = cost_A / delta_A      # $305
$_per_durable_B = cost_B / delta_B      # $255
